# 系统能力清单（System Capability Manifest）
# 实践场景学习平台

> **文档用途**：供 AI 理解本系统的能力边界、实体模型、操作规则和集成关系。
> 非 UI 规格说明，非业务流程手册。
> **版本**：v2.0.1

## 一、系统定位与边界

### 1.1 系统定位

面向职业院校的**场景化教研管理与学习测评系统**，核心价值是：

- **场景生产**：教研管理员/教师构建面向真实岗位场景的实践任务链，配置学习资源与测评规则
- **场景消费（教师端）**：组织学生完成场景测评，对学生提交进行评分管理
- **场景消费（学生端）**：在结构化场景中完成学习和测评，查看成绩

### 1.2 系统边界

| 边界项 | 说明/负责方 |
|--------|-----------|
| 岗位与能力点数据 | 本系统维护岗位-能力点映射关系，支持按专业分组管理 |
| 颗粒课数据 | 本系统维护颗粒课列表，知识点可关联颗粒课 |
| 试卷/题库 | 本系统维护试卷库与题库，支持选题组卷 |
| 排课系统 | 本期不做排课功能，默认固定账号查看场景 |
| 能力分析报告（学生端） | 本期教师端可查看学生能力报告，学生端暂不开放能力画像 |
| 任务分支/条件跳转 | 任务链仅支持顺序罗列，暂不支持分支 |
| 多版本数据隔离 | 已发布场景更新版本后，清除学生学习进度重新关联（不做版本快照存储） |

## 三、核心实体与状态

### 3.1 场景（Scene）

场景是平台的核心资源单元，由基础信息 + 任务链两层组成。

#### 基础信息字段

| 字段 | 说明 |
|------|------|
| 场景名称 | 必填 |
| 场景编码 | 系统自动生成（`SC-YYYY-XXXX`），不可修改 |
| 版本号 | 系统自动管理，每次编辑后版本升级 |
| 目标岗位 | 非必填，关联岗位列表；选中后自动填充"适用专业" |
| 所属批次 | 非必填，关联批次分组 |
| 面向行业 | 多选标签（互联网/IT、电子商务、金融、制造业、服务业） |
| 适用专业 | 多选标签（可从岗位自动填充，也可手动配置） |
| 难度等级 | 1-5 星（对应：入门/基础/中级/高级/专家） |
| 场景介绍 | 文本（背景/意义/学习目标） |
| 场景封面 | 图片上传，支持裁切 |
| 创建人 | 只读 |
| 共建人/共建部门 | 从组织架构选择，支持多选；有共建人时场景自动标记为"共建状态" |

#### 场景状态流转

```
草稿(draft)
  ↓ 提交审批
审批中(pending)
  ↓ 审批通过          ↓ 审批驳回
待发布(approved)    已驳回(rejected)
  ↓ 发布                ↓ 编辑 → 草稿(draft)
已发布(published)
  ↓ 取消发布
待发布(approved)
```

**各状态可执行操作**：

| 状态 | 可执行操作 |
|------|----------|
| 草稿(draft) | 编辑、删除、克隆、提交审批 |
| 审批中(pending) | 撤回审批（→草稿）、删除、克隆 |
| 待发布(approved) | 编辑（→草稿）、发布、删除、克隆 |
| 已驳回(rejected) | 编辑（→草稿）、删除、克隆 |
| 已发布(published) | 取消发布（→待发布）、克隆 |

**业务规则**：
- R-S01：任意状态均可克隆；克隆产生新场景，状态为草稿，名称默认为 `{原名称}(克隆)`
- R-S02：已发布场景不可直接删除；需先取消发布，再编辑或删除
- R-S03：场景版本更新（重新发布新版本）后，清除相关学生学习进度，学生重新关联新版本学习
- R-S04：批次分组调整不触发重新审批，可随时调整

---

### 3.2 任务（Task）

任务是场景的最小执行单元，场景下支持多个任务按顺序排列组成任务链。

#### 基础字段

| 字段 | 说明 |
|------|------|
| 任务名称 | 必填 |
| 任务编码 | 系统自动生成 |
| 任务类型 | 训练任务（`training`）/ 考核任务（`assessment`） |
| 预估学时 | 数字，学生完成任务的预估时长 |
| 难度 | 1-5 星 |
| 背景介绍 | 纯文本 |
| 详细描述 | 长文本，含任务目标/交付要求/测评要求等结构化内容 |
| 依赖任务 | 可配置前置依赖任务（数据记录，本期无实际前置校验） |

#### 任务链规则

- R-T01：任务链仅支持顺序罗列，不支持分支/条件跳转
- R-T02：任务顺序以箭头连接展示，支持在编辑界面调整顺序
- R-T03：所有任务权重之和必须等于 100%；不满足时系统黄色警告提示
- R-T04：引用任务（非克隆）的所有配置卡片（除权重外）为只读，不可编辑

#### 任务引入方式

| 方式 | 说明 |
|------|------|
| 新建任务 | 在当前场景下直接创建 |
| 克隆任务 | 从其他场景/自己的任务库中克隆，克隆后独立可编辑 |
| 引用任务 | 引用他人任务，只读，权重可独立配置 |

---

### 3.3 任务的 8 个配置卡片

每个任务由 8 个独立配置卡片组成，每张卡片独立保存：

| 卡片序号 | 卡片名称 | 核心字段 |
|---------|---------|---------|
| 1 | 任务基础信息（info） | 名称/类型/预估学时/难度/背景 |
| 2 | 任务说明（description） | 长文本任务说明书 |
| 3 | 考查知识点（knowledge） | 从知识点库中引用/克隆知识点；知识点可关联颗粒课 |
| 4 | 考查能力点（ability） | 从岗位能力模型中选择能力点；**必须先关联岗位才能配置此卡** |
| 5 | 任务资源（resources） | 支持 11 种资源类型：文档/表格/图片/链接/音频/视频/压缩包/场地/设施设备/软件/其他 |
| 6 | 测评形式（evaluation） | 从 4 种已开通测评形式中选择（可多选） |
| 7 | 评价规则（evaluationRules） | 针对已选测评形式分别配置：测评对象/评分主体/测评资源/评价标准 |
| 8 | 任务权重（weight） | 只读展示百分比；在"配置任务权重"弹窗/总评规则页面中统一编辑 |

**业务规则**：
- R-C04：卡片 4（考查能力点）在未关联岗位时，展示警告提示，不可配置
- R-C06：卡片 6（测评形式）本期仅开通 4 种：**现场问答、现场评审、试卷、题库**；其余形式展示"未开通"水印

---

### 3.4 知识点（Knowledge）

| 字段 | 说明 |
|------|------|
| 知识点名称 | 必填 |
| 描述 | 文本 |
| 编码 | 新建时自动生成，编辑时可修改 |
| 关联颗粒课 | 多选，来源于本系统颗粒课库 |

**知识点来源**：
- **引用**：引用公共知识点库中已有知识点（只读，标注"引用"角标）
- **克隆**：基于已有知识点创建副本，可编辑
- **新增**：在当前任务上下文中直接创建

### 3.5 资源（Resource）

11 种资源类型，字段按类型有差异：

| 资源类型 | 特有字段 |
|---------|---------|
| 文档/表格/图片/音频/视频/压缩包/其他 | 文件上传（最大 100MB） |
| 链接 | URL 地址 |
| 场地 | 场地地址/开放时间/容纳人数/联系人/联系电话 |
| 设施设备 | 所在位置/数量 |
| 软件 | 版本号/下载链接/授权信息 |

**公共字段**：资源名称（必填）、资源描述

### 3.6 四种测评形式的规则结构

#### 测评形式 1：现场问答（random_draw）

| 配置项 | 说明 |
|--------|------|
| 题目来源 | 从系统题库中选取 |
| 抽题规则 | 题目数量/难度分布/是否自动抽题 |
| 评价标准 | 配置评价量规或评分规则（与现场评审共用一套配置结构） |

#### 测评形式 2：现场评审（review）

| 配置项 | 说明 |
|--------|------|
| 材料要求 | 评审场地/环境资源准备/提交材料要求/截止天数/是否需要提交/允许重复提交 |
| 评审步骤 | 默认三步：初评/复评/终评；每步可配置名称/描述/评审主体/权重；支持自定义增删改 |
| 评价标准 | 配置评价量规或评分规则（与现场问答共用一套配置结构） |

#### 测评形式 3：试卷（paper）

| 配置项 | 说明 |
|--------|------|
| 试卷来源 | 从系统试卷库选择（展示试卷名称/题数/总分） |
| 新建试卷 | 支持在系统内配置试卷（选题组卷） |
| 参数配置 | 考试时长/及格分数/是否允许重考/重考次数/是否乱序/是否展示成绩 |
| 激活方式 | 手动 / 定时 / 永久开放 |

**说明**：试卷自动记分，不需要配置评价标准。

#### 测评形式 4：题库（question_bank）

| 配置项 | 说明 |
|--------|------|
| 题目来源 | 从系统题库选择；每道题可单独配置分值 |
| 抽题规则 | 题目数量/难度分布/总分/是否自动生成 |
| 参数配置 | 时间限制/题型权重/是否允许重考/是否乱序/是否展示成绩 |

**说明**：题库自动记分，不需要配置评价标准。

### 3.7 评分主体结构（评价规则卡片内）

适用于需要人工评分的测评形式（现场问答、现场评审）。6 种评分主体，每种独立开关控制启用，启用后各自有参数：

| 评分主体 | 关键参数 |
|---------|---------|
| 教师 | 专业背景/评分人数/多人记分规则（平均/中位数/最高/最低）/权重占比/最低教龄 |
| 企业导师 | 专业领域/最低工作年限/岗位工作经历/评分人数/多人记分规则/权重占比 |
| 自评 | 是否需要反思/反思最低字数/权重占比 |
| 互评 | 互评人数/互评规则/是否匿名/权重占比 |
| AI 评价 | AI 模型/权重占比 |
| 服务对象 | 服务方式/样本量/权重占比 |

**业务规则**：
- R-E01：各评分主体权重占比之和须等于 100%
- R-E02：测评对象（个人/小组）全局一致，本期无实际功能区分，仅作数据记录

---

### 3.8 评价标准（EvalPoint）结构

适用于：现场问答、现场评审（试卷和题库自动记分，不需要配置评价标准）。两种类型：

#### 类型 1：评价量规

| 构成要素 | 说明 |
|---------|------|
| 评价维度 | 一句话描述评价维度，可配置多条 |
| 关联能力点 | 多选，来自任务已配置的能力点 |
| 关联知识点 | 多选，来自任务已配置的知识点 |
| 等级转换规则 | 每个等级含：名称（如 A/B/C/D）/颜色/分数区间（min-max）/等级备注 |
| 维度权重 | 每条评价维度配置权重，加和须等于 100% |

#### 类型 2：评分规则

| 构成要素 | 说明 |
|---------|------|
| 评价项 | 一句话描述评价标准，可配置多条 |
| 加减分规则 | 一句话说明加减分打分参考 |
| 分值 | 每条评价项配置分值，支持一键均分（总分 100 分） |

**模板功能**：评价量规和评分规则均可保存为模板，支持一键覆盖、删除模板。

### 3.9 批次分组（Batch）

用于对场景进行分组管理，不影响场景内容，不触发重新审批（R-S04）。

| 字段 | 说明 |
|------|------|
| 分组名称 | 必填 |
| 批次编号 | 系统自动生成（`BG-YYYY-XXXX` 或 6 位随机字符），不可修改 |
| 关联审批流程 | 必填，关联一个审批流程模板 |

---

### 3.10 审批流程模板（ApprovalTemplate）

教研管理员预设的审批流模板，供批次关联使用。

| 字段 | 说明 |
|------|------|
| 流程名称 | 必填 |
| 流程说明 | 文本 |
| 审批步骤 | 动态列表，每步含：步骤名称/审批角色；至少保留 1 个步骤 |

**业务规则**：
- R-AP01：场景无批次时，提交审批时弹窗手动选择审批流程
- R-AP02：场景有批次时，使用批次关联的审批流程

## 四、操作能力清单

### 4.1 场景生产（教研管理员/教师）

| 操作 | 触发角色 | 前置条件 | 产出/结果 |
|------|---------|---------|---------|
| 新建场景 | 教研管理员/教师 | — | 场景对象创建，状态=草稿，生成场景编码 |
| 编辑基础信息（步骤1） | 创建者/共建人 | 场景状态 ≠ 已发布 | 场景基础信息更新 |
| 配置任务链（步骤2） | 创建者/共建人 | 场景状态 ≠ 已发布 | 任务链更新 |
| 添加任务 | 创建者/共建人 | — | 新任务附加到任务链末尾 |
| 克隆/引用任务 | 创建者/共建人 | 目标任务存在 | 克隆：新建可编辑副本；引用：只读关联 |
| 配置任务权重 | 创建者/共建人 | 任务数量≥1 | 各任务权重更新（加和必须=100%） |
| 配置成绩等级映射 | 创建者/共建人 | — | A/B/C/D 各等级分数区间、颜色、备注更新 |
| 上传/新增资源 | 创建者/共建人 | — | 资源进入资源库，并可关联至任务 |
| 提交审批 | 创建者（状态=草稿） | 场景名称非空 | 场景状态 → 审批中；进入审批队列 |
| 撤回审批 | 创建者（状态=审批中） | — | 场景状态 → 草稿 |
| 克隆场景 | 任意有权限用户 | 场景存在 | 新建草稿场景，名称=`{原名称}(克隆)` |
| 调整批次分组 | 创建者/共建人 | — | 场景归属批次更新，不触发重新审批 |
| 导入场景 | 教研管理员 | 文件格式为 xlsx/json | 批量创建草稿场景 |
| 导出场景 | 教研管理员/教师 | 场景已选中 | 导出为 Excel 或 JSON |

---

### 4.2 审批流程（教研管理员）

| 操作 | 触发角色 | 前置条件 | 产出/结果 |
|------|---------|---------|---------|
| 审核通过 | 有审批权限用户 | 场景状态=审批中 | 场景状态 → 待发布；可填写审批意见 |
| 审核驳回 | 有审批权限用户 | 场景状态=审批中 | 场景状态 → 已驳回；**驳回原因必填** |
| 发布场景 | 教研管理员 | 场景状态=待发布 | 场景状态 → 已发布；学生端可见 |
| 取消发布 | 教研管理员 | 场景状态=已发布 | 场景状态 → 待发布；学生端不可见 |
| 管理审批流程模板 | 教研管理员 | — | 新增/编辑/删除审批流程模板 |
| 管理批次分组 | 教研管理员 | — | 新增/编辑/删除批次 |

**业务规则**：
- R-PUB01：已发布场景重新编辑后进入草稿状态，需重新走审批流程才能发布新版本
- R-PUB02：发布新版本后，关联该场景的所有学生学习进度清零，重新关联新版本

---

### 4.3 场景消费——教师端（评分管理）

| 操作 | 触发角色 | 前置条件 | 产出/结果 |
|------|---------|---------|---------|
| 查看场景列表 | 教师 | — | 已发布场景列表（含待评分统计） |
| 查看任务列表 | 教师 | 场景已发布 | 各任务下学生提交状态统计 |
| 查看学生提交列表 | 教师 | 场景已发布 | 按入学年份与班级分组展示学生提交 |
| 评分——试卷/题库 | 教师 | 学生已提交 | 查看客观题自动评分，对主观题手动评分 |
| 评分——现场问答 | 教师 | 学生已参与现场 | 记录抽题情况，按评价点打分 |
| 评分——现场评审 | 教师 | 学生已提交材料 | 按评审量规多维度评价点打分 |
| 配置任务激活方式 | 教师 | 场景已发布 | 设置手动/定时激活，控制学生可见性 |
| 查看学生能力报告 | 教师 | 学生有提交记录 | 查看学生在该场景下的能力覆盖情况 |

### 4.4 场景消费——学生端

| 操作 | 触发角色 | 前置条件 | 产出/结果 |
|------|---------|---------|---------|
| 进入场景详情 | 学生 | 场景已发布 | 查看场景信息卡片/任务概览/资源中心/能力模型/评价标准/知识图谱 |
| 进入任务学习 | 学生 | 场景已发布 | 任务学习视图：查看任务说明/知识点/能力点/交付要求/配套资源 |
| 完成试卷测评 | 学生 | 任务配置了试卷测评 | 答题并提交，系统自动评分 |
| 完成题库测评 | 学生 | 任务配置了题库测评 | 从题库抽题作答，提交后系统自动评分 |
| 参与现场问答 | 学生 | 任务配置了现场问答 | 按预约时间参与，教师现场抽题，教师手动打分 |
| 提交评审材料 | 学生 | 任务配置了现场评审 | 上传成果材料，等待教师按评审量规打分 |
| 提交任务 | 学生 | 各测评形式均已完成 | 任务进入"待评分"状态；**提交后不可修改** |
| 查看成绩 | 学生 | 教师已完成评分 | 查看各测评形式得分与总分 |

**业务规则**：
- R-ST01：学生任务提交后进入"待评分"状态，**不可再次修改**
- R-ST02：现场问答和现场评审须教师手动评分；试卷和题库系统自动评分（客观题），主观题须教师复核
- R-ST03：本期不提供学生能力分析报告（学生不可查看能力画像）
