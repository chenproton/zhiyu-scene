## 1. 引言与术语

### 1.1 文档目的

本文档从**配置端**、**学生端**、**教师端**三个视角，完整描述 7 种测评方式的功能需求、交互流程、数据结构及状态流转，作为产品、设计与开发的对齐基准。

### 1.2 核心术语

| 术语 | 说明 |
|------|------|
| **场景（Scenario）** | 一个完整的教学/实训单元，包含岗位、批次、难度、任务链等 |
| **任务（Task）** | 场景下的一个学习节点，可配置多种测评方式 |
| **测评方式** | 对学生的学习成果进行测量的具体形式，共 7 种 |
| **评价点（EvalPoint）** | 评分维度，包含名称、权重、评分方式、等级映射等 |
| **评价主体** | 执行评分的人/角色，如教师、企业导师、学生互评、AI 等 |
| **量规（Rubric）** | 预定义的评价点模板，可一键引用到测评方式中 |

### 1.3 测评方式标识

| 显示名称 | 标识符 | 评价大类 |
|----------|--------|----------|
| 题库 | `question_bank` | 知识评价 |
| 试卷 | `paper` | 知识评价 |
| 随堂测 | `quiz` | 知识评价 |
| 现场问答 | `random_draw` | 过程评价 |
| 在线评审 | `review` | 成果评价 |
| 成果评价 | `outcome` | 成果评价 |
| 作业 | `homework` | 成果评价 |

## 2. 七种测评方式总览

### 2.1 分类体系

```
测评方式
├── 知识评价（客观/标准化，可自动判分）
│   ├── 题库
│   ├── 试卷
│   └── 随堂测
├── 过程评价（现场交互，口头/实操）
│   └── 现场问答
└── 成果评价（提交材料，主观评分）
    ├── 在线评审
    ├── 成果评价
    └── 作业
```

### 2.2 各端支持矩阵

| 测评方式 | 配置端 | 学生端 | 教师端 | 自动判分 | 支持评价主体 |
|----------|--------|--------|--------|----------|--------------|
| 题库 | ✓ | ✓ | ✓ | ✓ | 教师、企业导师 |
| 试卷 | ✓ | ✓ | ✓ | ✓ | 教师、企业导师 |
| 随堂测 | ✓ | ✓ | ✓ | ✓ | 教师、企业导师 |
| 现场问答 | ✓ | ✓ | ✓ | ✗ | 教师、企业导师、自评、互评 |
| 在线评审 | ✓ | ✓ | ✓ | ✗ | 教师、企业导师、自评、互评、服务对象 |
| 成果评价 | ✓ | ✓ | ✓ | ✗ | 教师、企业导师 |
| 作业 | ✓ | ✓ | ✓ | ✗ | 教师、企业导师 |

### 2.3 核心差异点

| 维度 | 知识评价（3种） | 过程评价（1种） | 成果评价（3种） |
|------|----------------|----------------|----------------|
| **学生交互** | 选择/输入答案 | 口头回答草稿 | 上传文件+文字 |
| **判分方式** | 客观题自动+主观题人工 | 完全人工 | 完全人工 |
| **时间限制** | 有（分钟级） | 现场决定 | 按截止日 |
| **重考/重交** | 可配置允许重考 | 一般不允许 | 可配置允许重交 |
| **材料提交** | 不需要 | 不需要 | 需要 |
| **多阶段评审** | 不支持 | 不支持 | 在线评审支持 |

## 3. 配置端需求：场景任务链配置

> 对应页面：`/scenarios/[id]/edit/tasks`

### 3.1 页面布局

采用**横向滚动卡片式布局**：
- 每行代表一个任务，支持拖拽排序
- 每行包含 8 张配置卡片：基础信息 / 任务说明 / 考查知识点 / 考查能力点 / 任务资源 / 测评形式 / 评价规则 / 任务权重
- 点击卡片弹出抽屉/弹窗进行详细配置

### 3.2 通用配置逻辑

#### 3.2.1 测评方式选择
- 在「测评形式」卡片中，以多选形式选择本任务需要的测评方式
- 选择后，「评价规则」卡片中显示对应测评方式的配置入口
- 每种测评方式在任务中占一定权重，所有方法权重之和必须等于 100%

#### 3.2.2 评价点配置（EvalPoint）

所有 7 种测评方式共用一套评价点配置体系：

**评价点字段**：
- `id`：唯一标识
- `name`：评价点名称（如"架构理论知识"、"沟通表达能力"）
- `desc`：描述说明
- `subType`：细分类型标签（知识掌握、操作规范、任务完成度、成果质量、沟通表达、协作能力、职业素养、创新能力、应变能力）
- `types`：多类型标签（支持一个评价点关联多种细分类型）
- `knowledgePointIds`：关联知识点 ID 列表
- `abilityPointIds`：关联能力点 ID 列表
- `scoringMethod`：评分方式
  - `score`：分值制（0-100 标准分）
  - `level`：等级制（A/B/C/D 四档）
  - `rubric`：量表制（预定义量规）
- `gradeMapping`：等级转换规则（各等级对应的分数区间和描述）
- `weight`：该评价点在所属测评方式内的权重

**评价点添加方式**：
1. 从能力点自动创建（关联能力点库）
2. 从知识点自动创建（关联知识点库）
3. 手动空白添加

**评价点配置面板**：
- 按细分类型分组展示（可折叠/展开）
- 每个评价点卡片支持：编辑名称、选择细分类型标签、选择评分方式、配置等级规则、关联知识点/能力点、设置权重

#### 3.2.3 量规引用
- 系统预定义 5 套量规方案：
  1. 前端开发能力评价量规
  2. 通用评审量规
  3. 创新实践评价量规
  4. 代码规范评分规则
  5. 出勤表现评分规则
- 每种测评方式可选择引用量规，引用后自动填充评价点列表

#### 3.2.4 评价主体配置

每种测评方式可配置的评价主体：

| 评价主体 | 题库/试卷/随堂测 | 现场问答 | 在线评审 |
|----------|------------------|----------|----------|
| 教师 | ✓ | ✓ | ✓ |
| 企业导师 | ✓ | ✓ | ✓ |
| 学生自评 | ✗ | ✓ | ✓ |
| 学生互评 | ✗ | ✓ | ✓ |
| AI 评价 | ✗ | ✗ | ✓ |
| 服务对象 | ✗ | ✗ | ✓ |

每个评价主体可配置参数：
- 权重占比
- 评分人数（如互评需指定 peerCount）
- 是否匿名
- 专业背景要求（如企业导师需指定 expertise）
- AI 模型选择及置信度阈值

### 3.3 各测评方式详细配置

#### 3.3.1 题库（`question_bank`）

**分类**：知识评价

**配置字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 题目选择 | `string[]` | 是 | 从题库中选择题目，左侧题库列表（我的/共建/公共），右侧已选列表 |
| 题目分值 | `Record<string, number>` | 是 | 每道已选题目的单独分值 |
| 随机抽题数量 | `number` | 否 | 如启用随机抽题，指定抽题数量 |
| 难度分布 | `enum` | 否 | 简单为主 / 难易混合 / 困难为主 |
| 时间限制 | `number` | 否 | 答题时限（分钟） |
| 允许重复测评 | `boolean` | 否 | 是否允许学生多次答题 |
| 最多重考次数 | `number` | 否 | 允许重考的最大次数 |
| 题目乱序 | `boolean` | 否 | 是否对题目顺序随机打乱 |
| 提交后展示成绩 | `boolean` | 否 | 提交后是否立即显示得分 |
| 评价点配置 | `EvalPoint[]` | 是 | 本测评方式的评价维度 |

**交互逻辑**：
- 使用题目选择器组件（QuestionSelectorPanel）进行题目选择
- 支持新增题目：单选、多选、判断、简答、论述、填空
- 已选题目支持拖拽排序、删除、修改分值
- 随机抽题与手动选题互斥

---

#### 3.3.2 试卷（`paper`）

**分类**：知识评价

**配置字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 试卷选择 | `string[]` | 是 | 从试卷库中选择（可多选），显示题数和总分 |
| 试卷权重 | `Record<string, number>` | 条件 | 多选试卷时，各试卷的得分权重，总和必须等于 100% |
| 考试时长 | `number` | 是 | 答题时限（分钟） |
| 及格分数 | `number` | 否 | 及格线分数 |
| 允许重考 | `boolean` | 否 | 是否允许重考 |
| 最多重考次数 | `number` | 否 | 最大重考次数 |
| 题目乱序 | `boolean` | 否 | 是否打乱题目顺序 |
| 交卷后显示成绩 | `boolean` | 否 | 提交后是否展示成绩 |
| 启用条件 | `enum` | 否 | 后台手动启用 / 定时启用 / 随时作答 |
| 定时启用时间 | `datetime` | 条件 | 启用条件为"定时启用"时必填 |
| 评价点配置 | `EvalPoint[]` | 是 | 本测评方式的评价维度 |

**交互逻辑**：
- 支持新建试卷：输入名称、题数、总分
- 多试卷模式下必须配置权重分配面板，提供"平均分配"快捷按钮
- 权重实时校验，合计不等于 100% 时阻止保存
- 支持查看试卷详情（题目列表、答案、解析）

#### 3.3.3 随堂测（`quiz`）

**分类**：知识评价

**配置字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 题目选择 | `string[]` | 是 | 与题库共用题目选择器 |
| 题目分值 | `Record<string, number>` | 是 | 每题分值 |
| 随机抽题数量 | `number` | 否 | 随机抽题数量 |
| 难度分布 | `enum` | 否 | 简单为主 / 难易混合 / 困难为主 |
| 时间限制 | `number` | 否 | 答题时限（分钟） |
| 允许重复测评 | `boolean` | 否 | 是否允许重复答题 |
| 题目乱序 | `boolean` | 否 | 是否打乱顺序 |
| 提交后展示成绩 | `boolean` | 否 | 提交后是否显示成绩 |
| 评价点配置 | `EvalPoint[]` | 是 | 本测评方式的评价维度 |

**交互逻辑**：
- UI 与题库几乎一致，但独立存储字段（`quizQuestions`）
- 与题库的区别：随堂测更轻量，通常题量更少，集成在学生端同一页面直接答题，不跳转外部系统

#### 3.3.4 现场问答（`random_draw`）

**分类**：过程评价

**配置字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 现场问答题库 | `Question[]` | 是 | 自定义现场问答题目列表 |
| 已选题目 | `string[]` | 是 | 从题库中勾选进入抽题池的题目 ID |
| 评价点配置 | `EvalPoint[]` | 是 | 评分维度 |
| 评分方式 | `enum` | 是 | 评价点评分 / 能力等级评定 |
| 引用量规 | `string` | 否 | 引用的量规方案 ID |

**现场问答题结构**：
- `id`：唯一标识
- `name`：题目名称
- `description`：题目描述/内容
- `answer`：参考答案要点
- `major`：适用专业（前端开发、后端开发、运维部署、通用）

**交互逻辑**：
- 左侧展示全部现场问答题表格（名称、适用专业、操作按钮）
- 右侧展示已配置的题目池
- 支持新增/编辑/删除/查看详情现场问答题
- 题目支持批量选择/取消
- 适用专业用于过滤不同专业学生看到的题目范围

#### 3.3.5 在线评审（`review`）

**分类**：成果评价

**配置字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 需要提交评审材料 | `boolean` | 否 | 学生是否需要上传材料 |
| 预估提交天数 | `number` | 条件 | 需要提交材料时，建议学生完成天数 |
| 提交材料要求描述 | `string` | 条件 | 对提交材料的内容、格式等要求 |
| 评审场地/环境资源 | `string` | 否 | 评审所需的场地、设备等准备说明 |
| 允许重新提交 | `boolean` | 否 | 是否允许学生修改后重新提交 |
| 评审流程步骤 | `ReviewStep[]` | 是 | 多阶段评审流程配置 |
| 评价点配置 | `EvalPoint[]` | 是 | 评分维度 |

**评审流程步骤结构（ReviewStep）**：
- `id`：步骤标识
- `label`：步骤名称（如"初评"、"复评"、"终评"）
- `desc`：步骤描述
- `enabled`：是否启用该步骤
- `subjectType`：评价主体（教师/企业导师/互评/自评/AI评价/服务对象）
- `weight`：该步骤在总分中的权重

**默认评审流程**：
1. 初评（教师，权重 40%）
2. 复评（专家组，权重 30%）
3. 终评（答辩委员会，权重 30%）

**交互逻辑**：
- 支持增删改评审步骤
- 每个步骤可独立设置启用状态、评价主体、权重
- 提供"一键平均分配权重"按钮
- 权重合计必须等于 100%，实时校验
- 未启用的步骤在评审流程中跳过

## 4. 学生端需求：学习与测评

> 对应页面：`/student.html`（学生端学习主界面）

### 4.1 页面架构

**整体布局**：
- 顶部：场景标题、进度概览
- 左侧：任务列表（可折叠）
- 右侧主区域：当前选中任务的详细内容

**学习模式选择**：
学生首次进入任务时，弹出模式选择弹窗：
1. **学测一体**：先学习任务内容（说明、资源），完成后直接进入测评
2. **仅学习**：只浏览任务说明和资源，暂不参与测评
3. **仅测评**：跳过学习环节，直接进入测评答题

### 4.2 任务状态体系

| 状态 | 含义 | 按钮显示 |
|------|------|----------|
| `not_started` | 未开始 | "开始学习" |
| `in_progress` | 学习中（进度 < 100%） | "继续学习 (x%)" |
| `pending` | 学习完成，待提交测评 | "提交任务" |
| `submitted` | 已提交，待评分 | "待评分"（禁用） |
| `graded` | 已评分 | "已评分 / 查看结果" |

### 4.3 多测评方式组合展示

当任务配置了多种测评方式时，采用 **2 列网格布局**：
- 每种测评方式一个独立卡片
- 顶部显示：图标 + 名称 + 权重百分比（如"题库 · 30%"）
- 卡片内展示该测评的专属交互界面
- 底部统一提交栏：显示各步骤完成检查清单 + 进度条 + 提交按钮

### 4.4 各测评方式学生端交互

#### 4.4.1 题库（`question_bank`）

**展示形式**：
- 绿色标签 `#52c41a`
- 显示题目总数、总分、权重
- 折叠/展开界面

**交互流程**：
```
折叠状态（显示概览：题数、总分、权重）
  → 点击"开始答题"
  → 弹出提示："请新开标签页开始答题"
  → 跳转外部测评系统答题
  → 答题完成后返回，状态更新
```

**状态管理**：
- 答题状态由外部系统管理，本页面仅展示入口和最终成绩回显
- 如配置"提交后展示成绩"，外部系统完成回传后显示得分

---

#### 4.4.2 试卷（`paper`）

**展示形式**：
- 蓝色标签 `#1677ff`
- 支持多张试卷列表展示
- 每张试卷显示：名称、题数、总分、权重
- 如配置定时启用，显示启用时间倒计时

**交互流程**：
```
折叠状态（显示试卷列表）
  → 点击单张试卷的"开始答题"
  → 弹出提示："请新开标签页开始答题"
  → 跳转外部测评系统
  → 多试卷模式下每张试卷独立显示状态和入口
```

**特殊状态**：
- 未启用（后台手动/定时）：显示"暂未开放"，按钮禁用
- 已启用：显示"开始答题"
- 已完成：显示得分和"已完成"标签

---

#### 4.4.3 随堂测（`quiz`）

**展示形式**：
- 红色标签 `#f5222d`
- **直接在本页面展开答题**，无需跳转外部系统
- 支持题型：单选、多选、判断

**交互流程**：
```
折叠状态（显示概览）
  → 点击"开始答题"
  → 展开所有题目（单选/多选/判断）
  → 点击选项选择答案
      - 单选：选中后高亮
      - 多选：可勾选多个，再次点击取消
      - 判断：是/否两个选项
  → 实时显示已答题目数（"已答 x/总题数"）
  → 点击"确认答案"提交
  → 显示正误反馈：
      - 正确选项：绿色高亮
      - 错误选项（如选了）：红色高亮
      - 正确答案：绿色高亮
  → 显示解析说明（如配置）
```

**状态管理**：
- `quizAnswers[taskId][questionId]`：存储用户选择的选项 ID
- `quizSubmitted[taskId]`：标记是否已提交
- 提交后选项变为只读，不可修改

**重考逻辑**：
- 如配置允许重复测评，提交后显示"重新答题"按钮
- 重新答题时清空上次答案，恢复可编辑状态

---

#### 4.4.4 现场问答（`random_draw`）

**展示形式**：
- 橙色标签 `#faad14`
- 显示"现场问答测评"通知卡片
- 展示评价点池（所有可能被抽中的题目，以 chip 标签展示）
- 显示评审信息：评审教师、场地、时间（占位或实际数据）

**交互流程**：
```
未抽题状态：
  → 显示题目池（所有候选题目标签）
  → 显示通知说明："现场问答将在实训现场由教师随机抽题进行"
  → 学生可预习题目池内容

教师现场抽题后（模拟/实际）：
  → 抽中题目高亮显示（.drawn 样式）
  → 展示"已抽中题目"结果卡片（题目内容、描述）
  → 提供作答草稿区（textarea 输入框）
  → 学生可输入思路并保存草稿
  → 点击"保存草稿"→ 存储到 sessionStorage
```

**状态管理**：
- `sessionStorage.setItem(`drawn-${taskId}`, idx)`：记录抽中题目索引
- `sessionStorage.setItem(`onsite-draft-${taskId}`, value)`：保存作答草稿

#### 4.4.5 在线评审（`review`）

**展示形式**：
- 紫色标签 `#722ed1`

**交互流程**：

**状态管理**：

#### 4.4.7 作业（`homework`）

### 4.5 统一提交机制

**提交前**：
- 底部固定提交栏显示各测评方式的完成检查清单
- 进度条显示整体完成度
- 未完成的测评方式以红色提示

**提交后**：

- 显示"已提交给教师评分"状态页

**提交校验**：
- 至少完成一种测评方式的必填内容
- 所有需要上传材料的测评方式必须已上传文件（或允许空提交时跳过）

## 5. 教师端需求：评分与审批

> 对应页面：`/approvals/grading`、`/approvals/grading/[id]`、`/approvals/grading/scenario/[scenarioId]/task/[taskId]`

### 5.1 页面架构

**场景任务评价首页**：
- 左侧：场景树导航（场景 → 任务分组）
- 右侧：任务及学生提交列表
- 支持按入学年份和班级分组筛选

**任务学生列表页**：
- 展示某场景下某任务的所有学生提交
- 按届/班级分组展示
- 每行显示：学生姓名、提交时间、状态（待评分/已评分）、操作按钮

**评分详情页（核心）**：
- 根据测评方式渲染不同的评分界面
- 通用结构：左侧学生提交内容，右侧评价点评分面板

---

### 5.2 评分状态体系

| 状态 | 含义 | 操作 |
|------|------|------|
| `pending` | 学生已提交，待教师评分 | 显示"评分"按钮，可进入评分页 |
| `graded` | 已评分 | 显示"已评分"标签，评分页只读 |

**评分提交后不可修改**：
- `handleSubmitGrading` 将 `status` 设为 `"graded"`
- 记录 `gradedAt`（评分时间）和 `gradedBy`（评分人）
- 界面进入只读状态，所有输入框禁用

---

### 5.3 通用评分组件

#### 5.3.1 标准分输入（StandardScoreInput）
- 教师输入 0-100 标准分
- 系统自动换算为加权后的实际分：`computedActual = (standard / 100) * maxScore`
- 实时显示匹配的等级（如"A档·表现卓越"）
- 下方显示评语输入框（多行文本）

#### 5.3.2 评价点评分卡片
- 评价点名称 + 权重
- 评分方式对应的输入组件（分值输入 / 等级选择 / 量规勾选）
- 关联的知识点/能力点标签展示
- 评语输入区

#### 5.3.3 总分计算
- 自动汇总各评价点的实际得分 × 权重
- 顶部显示最终总分输入框，教师可手动覆盖自动计算的总分
- 显示得分率（总分/满分×100%）

---

### 5.4 各测评方式教师端评分流程

#### 5.4.1 题库 / 试卷 / 随堂测

**布局**：上下结构（非左右分屏）

**顶部区域**：
- 最终总分输入框（可手动覆盖）
- 分项得分汇总：客观题自动得分、主观题得分、综合评价

**题目区域**：
- 每道题目以折叠卡片展示
- 支持筛选："全部题目" / "待评分题目" / "客观题" / "主观题"

**客观题（单选/多选/判断）**：
- 自动判分，显示得分
- 学生答案 vs 正确答案对比
- 正确：绿色勾选；错误：红色叉号

**主观题（问答/简答/论述）**：
- 教师输入 0-100 标准分
- 系统自动换算为实际分
- 显示参考答案要点供教师对照

**综合评价点**：
- 试卷类型额外显示"综合评价点"评分卡片（如：架构理论知识、UML建模能力、数据库设计能力）
- 随堂测有独立的 `evalPoints.quiz` 配置

---

#### 5.4.2 现场问答（`random_draw`）

**布局**：左右分屏

**左侧**：
- 显示抽中的题目记录（题目内容、选项、参考答案）
- 教师可记录学生口头回答要点（textarea）
- 如有多轮抽题，以时间线形式展示每轮记录

**右侧**：
- 评价点评分卡片列表（`evalPoints.randomDraw`）
- 典型评价点：业务理解深度、方案论证能力、技术表达与沟通、临场应变能力

**评分流程**：
```
查看抽中题目和参考答案
  → 听取学生口头回答 / 查看回答记录
  → 对每个评价点输入标准分（0-100）
  → 系统自动计算实际分和等级
  → 输入评语
  → 确认总分
  → 提交评分
```

---

#### 5.4.3 在线评审（`review`）

**布局**：左右分屏

**左侧**：
- 评审进度条/步骤指示器（初评 → 复评 → 终评）
- 当前阶段高亮显示
- 上一阶段评价历史（可折叠展开查看）
- 学生提交的文本答案（只读展示）
- 附件列表：支持预览（代码/视频/文档/图片）和下载

**右侧**：
- 评价点评分卡片列表（`evalPoints.review`）
- 典型评价点：文档规范性、设计合理性、数据库设计质量、接口设计规范、代码规范性、功能实现完整性、安全设计意识、方案可维护性

**多阶段评审流程**：
```
确认当前评审阶段（初评/复评/终评）
  → 查看学生提交材料（文本+附件）
  → 如需参考，展开查看上一阶段评分结果
  → 对每个评价点评分
  → 系统根据阶段权重计算该阶段得分
  → 输入阶段评语
  → 提交本阶段评分
  → 如还有后续阶段，状态变为"等待下一阶段评审"
  → 全部阶段完成后，汇总为最终得分
```

**评价主体切换**：
- 不同评价主体（教师/专家/答辩委员会）登录后只能看到属于自己阶段的评分入口
- 管理员可查看全部阶段

---

#### 5.4.4 成果评价（`outcome`）

**布局**：左右分屏

**左侧**：
- 学生成果材料展示
- 文本答案只读展示
- 附件预览/下载（PDF/DOCX/MP4/ZIP）

**右侧**：
- 评价点评分卡片列表（`evalPoints.outcome`）
- 典型评价点：优化方案完整性、实施效果可量化、文档规范性、创新性

**评分流程**：
```
下载/预览学生成果材料
  → 阅读文本说明
  → 对每个评价点进行评分
  → 系统计算总分
  → 输入总体评语
  → 提交评分
```

---

#### 5.4.5 作业（`homework`）

**布局**：左右分屏

**左侧**：
- 学生作业材料展示
- 附件预览/下载

**右侧**：
- 评价点评分卡片列表（`evalPoints.homework`）
- 如未配置评价点，显示"暂无评价标准"占位提示

**评分流程**：
```
下载/预览学生作业
  → 如配置有评价点，逐项评分
  → 如未配置评价点，直接输入总分和评语
  → 提交评分
```

> **注意**：当前实现中作业类型可能未配置评价点，需要产品确认是否统一要求配置。

---

### 5.5 评分汇总与回显

**评分完成后**：
- 学生列表页该学生状态变为"已评分"
- 显示得分（如：85分）
- 点击可查看评分详情（只读）

**评分详情回显**：
- 各评价点得分、等级、评语
- 总分和得分率
- 评分人、评分时间

### 7.5 待确认事项

1. **作业评价点**：当前实现中作业类型可能未配置评价点，需确认是否统一要求所有测评方式都必须配置评价点。
2. **学生端模拟系统**：当前 simulation 目录下仅实现 4 种测评方式（试卷、题库、现场问答、评审），需确认是否补齐随堂测、作业、成果评价的模拟。
3. **外部系统对接**：题库和试卷的学生端答题跳转外部系统，需明确对接协议和数据回传机制。
4. **AI 评价主体**：在线评审支持 AI 评价，需明确 AI 评分触发条件和结果人工复核机制。
5. **互评/自评流程**：现场问答和在线评审支持学生互评/自评，需明确互评的分配算法和防作弊机制。
