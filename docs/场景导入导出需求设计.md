# 场景导入导出需求设计文档（定稿版）

> 最后更新：2026-05-18  
> 适用范围：场景大厅页面的场景批量导入与导出功能  
> **导出格式：仅 Excel（.xlsx）**  
> **导入格式：仅 Excel（.xlsx）**

---

## 1. 概述

本文档定义「场景大厅」页面中**场景导入**与**场景导出**功能的业务逻辑、数据结构和交互规则。

- **导出**：支持将选中的场景（含场景下全部任务信息）导出为 Excel 文件。
- **导入**：支持通过 Excel 文件批量导入场景数据，需处理与现有数据的冲突（覆盖/新建/跳过）。
- **不涉及 JSON**：本期导入导出均只支持 Excel 格式，JSON 导入导出暂不做。

---

## 2. 场景导出

### 2.1 功能入口
- **位置**：场景大厅 → 批量操作栏 →「导出」按钮
- **前置条件**：列表中至少选中 1 个场景
- **批量行为**：导出对话框显示「已选择 N 个场景」，点击确认后直接下载 `.xlsx` 文件

### 2.2 导出文件结构（多 Sheet）

Excel 文件包含 **2 个 Sheet**，分别存放场景信息和任务信息。

### 2.3 单元格内 JSON 字段规范

以下字段在 Excel 中以 **JSON 字符串**形式保存在单个单元格内，确保复杂结构不丢失。

#### 场景列表中的 JSON 字段

**权重配置（`权重配置`列）**
```json
{
  "tasks": [
    { "taskId": "task-1-1", "weight": 20 },
    { "taskId": "task-1-2", "weight": 30 }
  ]
}
```

**等级映射（`等级映射`列）**
```json
[
  { "id": "grade-1", "grade": "A", "minScore": 90, "maxScore": 100, "color": "bg-green-500", "remark": "表现卓越" },
  { "id": "grade-2", "grade": "B", "minScore": 75, "maxScore": 89, "color": "bg-blue-500", "remark": "表现良好" }
]
```

#### 任务列表中的 JSON 字段

**测评配置（`测评配置`列）**
```json
{
  "type": "objective",
  "objectiveConfig": {
    "totalScore": 100,
    "questions": [
      { "id": "q-1", "type": "single", "content": "...", "options": ["A", "B"], "answer": "A", "score": 10 }
    ]
  }
}
```
或主观题：
```json
{
  "type": "subjective",
  "subjectiveConfig": {
    "totalScore": 100,
    "rubricPoints": [...],
    "synthesisRule": "weighted"
  }
}
```

**评价点配置（`评价点配置`列）**
```json
{
  "randomDraw": [
    { "id": "rep-1", "name": "理论知识掌握程度", "desc": "...", "weight": 40, "maxScore": 40, "scoringMethod": "level" }
  ],
  "review": [...],
  "paper": [...],
  "questionBank": [...]
}
```

**评审步骤（`评审步骤`列）**
```json
[
  { "id": "rs-1", "label": "初评", "desc": "由指导教师进行第一轮评审", "enabled": true, "subjectType": "teacher" }
]
```

**交付物配置（`交付物配置`列）**
```json
[
  { "id": "del-1", "type": "exercise", "name": "架构知识测验", "description": "...", "evaluationPoints": [] }
]
```

> **设计说明**：
> - 复杂嵌套结构以 JSON 字符串保存在单元格内，兼顾 Excel 的可读性和数据完整性。
> - 导入时解析这些 JSON 字符串，若格式非法则该字段置为空数组/null，并在导入结果中提示。
> - `测评配置` 列中 `type` 为 `null` 时，表示该任务无测评。

## 3. 场景导入

### 3.1 功能入口
- **位置**：场景大厅 → 顶部操作区 →「导入场景」按钮
- **支持格式**：仅 `.xlsx`
- **文件限制**：单个文件不超过 10MB，**单次最多导入 50 个场景**

### 3.2 导入流程

```
┌─────────────────────────────────────────┐
│  1. 用户上传文件（点击或拖拽）             │
│     → 校验格式（仅 .xlsx）                │
│     → 校验大小（≤ 10MB）                  │
│     → 校验场景数量（≤ 50 个）             │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  2. 前端解析 Excel 内容                  │
│     → 读取「场景列表」「任务列表」Sheet   │
│     → 解析单元格内 JSON 字符串            │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  3. 数据预校验                           │
│     → 必填字段检查（场景编码、场景名称）   │
│     → 场景数量上限检查（≤ 50）            │
│     → 任务序号唯一性检查                 │
│     → 关联关系一致性检查（任务所属场景）   │
│     → JSON 字段格式校验                  │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  4. 冲突检测                             │
│     → 按「场景编码」匹配现有数据           │
│     → 识别：新增场景 / 与现有场景冲突      │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  5. 冲突处理策略选择（弹窗确认）           │
│     → 覆盖现有：更新同名场景及任务        │
│     → 新建场景：完全重新生成编码          │
│     → 跳过：保留现有，不导入该场景        │
└─────────────────┬───────────────────────┘
                  ▼
┌─────────────────────────────────────────┐
│  6. 执行导入                             │
│     → 写入场景数据（状态强制为 draft）     │
│     → 写入任务数据                       │
│     → 更新 weightConfig、gradeMapping    │
│     → 提示导入结果（成功/失败/跳过统计）   │
└─────────────────────────────────────────┘
```

### 3.3 冲突检测规则

冲突以**场景编码（`code`）**为唯一匹配键：

| 场景编码状态 | 判定结果 | 处理方式 |
|-------------|---------|---------|
| 文件中编码不存在于当前系统 | **新增场景** | 直接导入，状态设为 `draft` |
| 文件中编码已存在于当前系统 | **冲突场景** | 弹出策略选择：覆盖 / 新建 / 跳过 |

> **注意**：场景名称不作为匹配键，因为名称可能重复；编码是系统生成的唯一标识。

### 3.4 冲突处理策略

当检测到冲突时，向用户展示冲突列表（场景编码 + 场景名称），并提供三种策略：

#### 策略 A：覆盖现有
- **行为**：用文件中的数据**完全替换**现有场景（含任务）。
- **适用范围**：**允许覆盖任意状态的场景**，包括 `published`。
- **影响**：
  - 场景基础信息更新。
  - 原任务全部删除，按文件重新创建任务。
  - 状态重置为 `draft`（即使原场景是 published，导入后也变为草稿，需重新提交审批）。
  - `id` 保持不变（覆盖更新），`updatedAt` 更新为当前时间。
- **风险提示**：覆盖后原场景的审批记录、学生提交数据等可能失效，需提示用户。

#### 策略 B：新建场景
- **行为**：保留现有场景不变，为导入数据**按系统规则完全重新生成场景编码**，创建为新场景。
- **编码生成规则**：
  - 完全重新按系统规则生成新编码（如 `SC-2026-00XX`），不保留原编码的任何部分。
  - `id` 也重新生成。
- **影响**：
  - 生成全新 `id` 和 `code`。
  - 状态为 `draft`。
  - 创建人默认为当前操作用户。
  - **任务编码同步重新生成**，避免与现有任务编码冲突。

#### 策略 C：跳过
- **行为**：不导入该场景，保留现有数据不变。
- **适用**：用户只想导入部分场景，或确认某场景无需更新。

### 3.5 导入数据规范

#### 通用规则
1. **状态强制重置**：无论文件中 `status` 字段是什么，导入后统一设为 `draft`。避免用户通过导入绕过审批流程直接发布场景。
2. **编码重新生成（新建策略下）**：
   - **场景编码**：按系统规则完全重新生成。
   - **任务编码**：同步重新生成，避免全局任务编码冲突。
3. **创建人/共建人处理**：
   - 若文件中创建人名称在当前系统存在，则关联该用户。
   - 若不存在，创建人设为当前操作用户，共建人列表清空（或仅保留存在的用户）。
4. **时间字段**：`createdAt` 设为导入时间，`updatedAt` 设为导入时间，`publishTime` 清空。
5. **批次/岗位关联**：
   - 按名称匹配现有批次和岗位。
   - 若匹配不到，字段置空（导入的场景成为无批次/无岗位场景，后续可手动编辑补充）。
6. **单次导入上限**：最多 50 个场景，超出时拦截并提示「单次最多导入 50 个场景，请分批导入」。

## 4. 边界情况与异常处理

| 场景 | 处理策略 |
|------|---------|
| 文件格式不正确（非 .xlsx） | 拦截上传，提示「仅支持 .xlsx 格式」 |
| 文件超过 10MB | 拦截上传，提示「文件大小超过 10MB 限制」 |
| 场景数量超过 50 个 | 拦截上传，提示「单次最多导入 50 个场景，请分批导入」 |
| Excel 缺少「任务列表」Sheet | 允许导入，仅导入场景基础信息，任务为空数组 |
| Excel 任务列表中存在无法匹配的场景编码 | 该任务标记为异常，导入结果中提示「X 个任务因所属场景编码未匹配而跳过」 |
| 单元格内 JSON 字符串格式非法 | 该字段置为空值（null / [] / {}），继续导入，结果中提示具体错误行 |
| 缺少必填字段（场景编码/名称） | 该场景标记为失败，继续导入其他场景 |
| 导入任务权重总和不等于 100% | 导入成功但提示警告「任务权重总和不等于 100%，请进入编辑器调整」 |
| 导入的场景编码与系统生成规则冲突 | 按「新建场景」策略重新生成编码 |

## 5. 权限与安全

1. **操作权限**：仅场景创建人、共建人、教研管理员可执行导入导出。
2. **状态约束**：
   - 导出：任意状态的场景均可导出。
   - 导入：导入后状态强制为 `draft`，无论原文件中状态如何。
3. **数据隔离**：导入的场景默认归属当前操作用户为创建人。
4. **覆盖 published 场景**：允许覆盖，但需在确认弹窗中明确提示风险。
