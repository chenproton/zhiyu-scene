# 实践场景学习平台 — 产品需求文档（PRD）

> 文档版本：v1.0  
> 面向对象：前端 / 后端 / 测试开发团队  
> 范围说明：本文档覆盖平台全部功能模块的产品需求与交互细节，不含数据模型与接口设计，不含非功能需求（性能、安全、部署）。

## 1. 产品概述

### 1.1 产品定位

「实践场景学习平台」是面向职业院校的场景化教研管理与学习测评系统。平台支持教研管理员/教师构建面向真实岗位场景的实践任务链，配置多元化的学习资源与测评形式，并基于岗位能力点体系对学生的场景完成度进行胜任度分析。

### 1.2 目标用户

| 角色 | 核心职责 | 使用场景 |
|------|---------|---------|
| 教研管理员 | 生产场景资源、管理审批流程与批次、维护岗位能力体系 | 场景编辑器、审批流程管理、批次管理、岗位能力点管理 |
| 职业院校教师 | 消费场景资源（组织测评、评分）、共建场景 | 场景大厅、审批中心、评分管理、学生能力报告 |
| 学生 | 消费场景资源（学习+测评） | 任务学习、在线测评、能力报告查看 |

### 1.3 核心工作流

1. **场景生产**：教研管理员/教师在「场景大厅」创建场景 → 编辑基础信息 → 配置任务链（每个任务配置8个步骤：基础信息/任务说明/知识点/能力点/资源/测评形式/评价规则/权重）→ 提交审批 → 审批通过后发布。
2. **场景消费（教师端）**：教师查看已发布场景 → 激活任务 → 查看学生提交 → 按评价规则评分。
3. **场景消费（学生端）**：学生进入场景任务 → 查阅学习资源与AI助手 → 完成测评（试卷/题库/现场问答/现场评审）→ 查看成绩与能力分析报告。

## 3. 全局交互规范

### 3.1 页面布局框架

- **顶部固定导航栏**：平台名称（实践场景学习平台）、全局搜索（如有）、用户头像/退出。
- **左侧固定侧边栏**：宽度 `w-56`（224px），顶部包含返回按钮+平台标题，下方为导航菜单，底部为版本信息卡片。
- **主内容区**：左侧留空 `pl-56`，顶部留空 `pt-14`，内容区最大宽度 `max-w-6xl` 或 `max-w-[1400px]` 居中。
- **场景编辑器特殊布局**：进入场景编辑（基础信息/任务链/总评规则）时，页面采用 `fixed inset-0 z-50` 全屏覆盖模式，隐藏侧边栏，顶部为 sticky 操作栏。

### 3.2 侧边栏导航项

| 导航项 | 路由 | 高亮规则 |
|-------|------|---------|
| 数据统计工作台 | `/dashboard` | pathname === `/dashboard` |
| 批次管理 | `/batches` | pathname.startsWith(`/batches`) |
| 审批流程管理 | `/workflows` | pathname.startsWith(`/workflows`) |
| 场景大厅 | `/` | pathname === `/` 或 `/scenarios/*` |
| 审批中心 | `/approvals` | `/approvals/*` 且不含 `/approvals/grading` |
| 场景任务评价 | `/approvals/grading` | `/approvals/grading/*` 且不含 `/grading/simulation` |
| 学生能力报告 | `/students` | pathname.startsWith(`/students`) |
| 岗位能力点管理 | `/positions` | pathname.startsWith(`/positions`) |

- 当前激活项样式：`bg-primary/5 text-primary border-l-2 border-primary -ml-[2px]`。
- 非激活项悬停：`hover:bg-gray-50 hover:text-gray-800`。

### 3.3 通用组件交互规范

| 组件 | 交互规则 |
|------|---------|
| Button（主操作） | `size="sm"` 为主，图标+文字组合。禁用态需置灰并禁用点击。 |
| Button（危险操作） | 删除、驳回等使用 `text-red-500 hover:text-red-600 hover:bg-red-50`。 |
| Card | 默认带 `border-slate-200 shadow-sm`，内容区用 `CardContent` 包裹。 |
| Dialog | 标准弹窗宽度 `sm:max-w-[450px]`~`sm:max-w-4xl`，头部 `DialogHeader`+`DialogTitle`+`DialogDescription`，底部 `DialogFooter`。点击遮罩或关闭按钮可关闭，表单未保存时建议二次确认（本期可先不做二次确认）。 |
| Tabs | 用于同页视图切换，如「我的场景/共建场景/公共场景」。 |
| Badge | 场景状态使用不同颜色：`draft(gray)`、`pending(yellow)`、`approved(blue)`、`rejected(red)`、`published(green)`。 |
| Table | 表头 `bg-slate-50`，文字 `text-xs font-medium text-slate-500`。操作列固定右侧，带左侧阴影 `shadow-[-4px_0_8px_-4px_rgba(0,0,0,0.05)]`。 |
| Select / Input | 表单字段统一使用 `Label` 标签，必填项标注红色星号 `*`。 |
| 空状态 | 统一使用：图标（灰色）+ 标题 + 描述 + 操作按钮（如有）。 |

### 3.4 状态流转（场景）

```
草稿(draft) → 提交审批 → 审批中(pending) → 审批通过(approved) → 发布(published)
                                      ↘ 驳回(rejected) → 修改后回到草稿
已发布 → 取消发布 → 回到草稿
```

- **草稿**：可编辑、可删除、可提交审批。
- **审批中**：不可编辑，可撤回审批回到草稿。
- **已通过**：不可编辑，可发布。
- **已驳回**：可编辑修改，重新提交审批。
- **已发布**：不可编辑，可取消发布回到草稿。

## 4. 模块详细需求


### 4.1 场景管理（场景大厅）

**路由**：`/`（首页）

**功能概述**：场景大厅是平台的资源管理中心，支持教研管理员和教师查看、筛选、管理自己创建/共建/公共的场景资源。支持列表视图和批次分组视图两种展示模式。

#### 4.1.1 页面布局

页面整体采用垂直堆叠布局，内容区宽度为全屏（无 `max-width` 限制，左右保留 `px-6` padding），从上到下依次为：

1. **顶部标题卡片**（`Card`）
2. **视图切换区**
3. **数据列表卡片**（含筛选栏+批量操作栏+场景列表）
4. **空状态区**（无数据时展示）

#### 4.1.2 顶部标题卡片

- **左侧**：
  - 标题：`场景管理`，`text-xl font-semibold text-slate-900`
  - 副标题：`维护场景信息、任务信息等场景资源管理功能`，`text-xs text-slate-500`
- **右侧操作按钮组**（从左到右）：
  1. **配置审批流程**（`variant="outline" size="sm"`，图标 `GitBranch`）：点击跳转 `/workflows`。
  2. **配置批次分组**（`variant="outline" size="sm"`，图标 `FolderKanban`）：点击打开「批次分组管理」弹窗。
  3. **导入资源包**（`variant="outline" size="sm"`，图标 `Upload`）：点击打开「导入资源包」弹窗。
  4. **导入场景**（`variant="outline" size="sm"`，图标 `Upload`）：点击打开「导入场景」弹窗（支持 Excel/JSON，上传区域为虚线边框拖拽区）。
  5. **新建场景**（`size="sm" className="bg-primary"`，图标 `Plus`）：点击跳转 `/scenarios/new/edit`。
- **统计面板**（仅在「我的场景」「共建场景」Tab 下显示）：
  - 5个 mini 卡片横向排列，每卡包含：指标名（`text-xs text-slate-500`）+ 数值（`text-xl font-bold text-slate-900`）+ 图标（圆形背景色）。
  - 指标：场景总数、未提交（draft）、审批中（pending）、已驳回（rejected）、已发布（published）。
  - 图标背景色：总数-blue、未提交-gray、审批中-yellow、已驳回-red、已发布-green。

#### 4.1.3 视图切换区

- **左侧 Tab**：`我的场景` / `共建场景` / `公共场景`（`Tabs` 组件，`grid-cols-3`）
  - 切换 Tab 时，清空已选项、岗位筛选、批次筛选。
  - 「我的场景」：当前用户为 `creatorId` 的场景，包含该用户所有非已发布状态的场景（草稿、审批中、已通过、已驳回）。
  - 「共建场景」：当前用户在 `coBuilders` 列表中的场景。
  - 「公共场景」：仅展示 `status === "published"` 的已发布场景，不展示统计面板。
- **右侧视图模式切换**：`资源列表` / `批次分组`（自定义按钮组，选中态 `bg-primary text-primary-foreground`）

#### 4.1.4 筛选栏

位于数据列表卡片内顶部：

- **搜索框**（`Input`，左侧 `Search` 图标）：placeholder `搜索场景名称 / 任务名称`，支持按场景名称和任务名称模糊搜索。
- **岗位筛选**（`Select`，宽度 `w-44`）：选项分组展示，先按专业（`profession.name`）分组，再展示岗位（`position.name`）。选项 `全部岗位`。
- **批次筛选**（`Select`，宽度 `w-44`）：选项为全部批次 + 各批次名称（附带 `code`）。
- **状态筛选**（`Select`，宽度 `w-36`）：全部状态 / 草稿 / 审批中 / 已通过 / 已驳回 / 已发布。
- **重置按钮**（`variant="outline" size="sm"`，图标 `RotateCcw`）：清空所有筛选条件。

#### 4.1.5 批量操作栏

位于筛选栏下方，带 `border-t border-slate-100 pt-2` 分隔：

- 左侧文案：`已选择 N 项：`（有选中时高亮 `text-slate-700 font-medium`）/ `请选择场景：`（无选中时置灰）。
- 操作按钮（`variant="outline" size="sm" className="h-8 text-xs"`，按业务逻辑启用/禁用）：
  1. **提交审批**（`Send`）：仅当选中项包含 `draft` 或 `rejected` 时可用。
  2. **撤回审批**（`Undo2`）：仅当选中项包含 `pending` 时可用。
  3. **发布**（`ArrowUpFromLine`）：仅当选中项包含 `approved` 时可用。
  4. **取消发布**（`ArrowDownFromLine`）：仅当选中项包含 `published` 时可用。
  5. **删除**（`Trash2`）：仅当选中项包含 `draft` 或 `rejected` 时可用。
  6. **克隆**（`Copy`）：有选中即可用。
  7. **调整批次分组**（`FolderKanban`）：有选中即可用（打开批次选择弹窗）。
  8. **导出**（`Download`）：有选中即可用（打开导出格式选择弹窗）。

> 批量操作执行后，清空选中态（`selectedIds`）。

#### 4.1.6 场景列表（资源列表视图）

使用 `ScenarioList` 组件，表格结构：

| 列 | 宽度 | 内容 |
|---|------|------|
| 复选框 | `col-span-1` 居中 | 单行复选 + 表头全选（含 `indeterminate` 态） |
| 场景名称 | `col-span-2` | 名称（`font-medium text-slate-900`，hover 变 `text-primary`）+ 状态 Badge |
| 场景编码 | `col-span-1` | `text-sm text-slate-600 truncate` |
| 版本 | `col-span-1` 居中 | `text-sm text-slate-600` |
| 所属岗位 | `col-span-1` | `text-sm text-slate-600 truncate` |
| 所属批次分组 | `col-span-2` | `text-sm text-slate-600 truncate` |
| 创建人 | `col-span-1` | `text-xs text-slate-500 truncate` |
| 发布时间 | `col-span-1` | `text-xs text-slate-500 truncate`，无则为 `-` |
| 场景任务数量 | `col-span-1` 居中 | 蓝色 Badge，点击跳转任务链配置 `/scenarios/{id}/edit/tasks` |
| 操作 | `col-span-1` 右对齐 | 悬浮操作栏（见下文） |

- **行交互**：hover 时行背景变 `hover:bg-slate-50`，选中行 `bg-primary/5`。
- **悬浮操作栏**：默认 `opacity-0`，hover 行时 `opacity-100`，带白色半透明背景+阴影。操作按钮从左到右：
  1. **查看详情**（`Eye`）：跳转 `/scenarios/{id}/edit`。
  2. **编辑**（`Pencil`）：跳转 `/scenarios/{id}/edit`。
  3. **编排任务**（`GitBranch`）：跳转 `/scenarios/{id}/edit/tasks`。
  4. **克隆场景**（`Copy`）：触发克隆确认弹窗。
  5. **提交审批**（`Send`）：仅草稿/驳回态显示。
  6. **撤回审批**（`Undo2`）：仅审批中态显示。
  7. **查看驳回原因**（`MessageSquare`）：仅驳回态显示，打开驳回原因弹窗。
  8. **删除**（`Trash2`）：红色文字。

#### 4.1.7 批次分组视图

当视图模式为 `group` 时：

- 按 `batchId` 分组展示，每个批次为一个 `Collapsible` 折叠面板。
- 面板头部：批次名称 + 编码 + 状态 Badge（`开放中`/`已截稿`）+ 场景数量。
- 面板内容：该批次下的场景列表（复用 `ScenarioList` 组件）。
- 无批次的场景单独放在底部「未分类」区域（虚线边框），不参与批次分组。

### 4.2 场景编辑器 — 基础信息

**路由**：`/scenarios/new/edit`（新建）或 `/scenarios/{id}/edit`（编辑）

**功能概述**：场景编辑器第一步，用于配置场景的基础元信息。页面采用全屏覆盖模式（`fixed inset-0 z-50`）。

#### 4.2.1 顶部操作栏（Sticky）

- **左侧**：
  - 「取消」按钮（`variant="ghost"`，图标 `X`）：点击返回场景大厅 `/`。
  - 分隔线（`h-5 w-px bg-gray-200`）。
  - 步骤标识：`Badge bg-primary text-primary-foreground` 显示「步骤 1」+ 文案「基础信息编辑」。
- **右侧**：
  - 「保存草稿」（`variant="outline" size="sm"`，图标 `Save`）。
  - 「预览」（`variant="outline" size="sm"`，图标 `Eye`）：打开预览弹窗。
  - 「下一步」（主按钮，图标 `ArrowRight`）：必填校验通过后可用，跳转 `/scenarios/{id}/edit/tasks`。

> 必填校验规则：`scenarioName` 非空且 `industryIds` 至少选择一项。

#### 4.2.2 页面主体（Grid 布局 `grid-cols-3 gap-6`）

**新建场景与编辑场景的差异化说明**：

| 差异点 | 新建场景 (`/scenarios/new/edit`) | 编辑场景 (`/scenarios/{id}/edit`) |
|-------|----------------------------------|-----------------------------------|
| 页面标题 | `新建实践场景` | `编辑实践场景` |
| 预填信息 | 顶部展示预填信息卡片（所属岗位、所属批次），从首页新建弹窗或 URL Query 参数带入，只读展示 | 无预填信息卡片 |
| 岗位/批次选择 | 已预填的岗位/批次直接锁定展示，不可在此页修改（如需修改需返回首页重新创建） | 可自由修改目标岗位和所属批次 |

### 4.3 场景编辑器 — 任务链配置

**路由**：`/scenarios/{id}/edit/tasks`

**功能概述**：场景编辑器第二步，以「任务链」形式顺序罗列场景下的所有任务。每个任务通过 8 个配置卡片来定义学习+测评过程。任务链仅支持简单的顺序罗列，暂不支持分支/条件跳转。

#### 4.3.1 顶部操作栏（Sticky）

- **左侧**：
  - 「返回」按钮（`variant="ghost"`，图标 `ArrowLeft`）：返回基础信息编辑页 `/scenarios/{id}/edit`。
  - 分隔线 + 步骤标识：「步骤 2」+「任务链配置」。
- **右侧**：
  - 「保存草稿」、「预览」、「完成配置」（主按钮，图标 `ArrowRight`）：点击返回场景大厅 `/`，同时保存当前任务链状态。

#### 4.3.2 场景信息摘要卡片

展示当前场景的关键信息：
- 场景名称（`CardTitle`）+ 共建 Badge（如有）。
- 岗位名 `|` 行业名 `|` 专业名 + 共建人列表（Tag 形式）。
- 右侧：难度星级 + 版本号 Badge。
- 底部 border-t：场景背景介绍文本。

#### 4.3.3 任务列表操作栏

- **左侧**：
  - 「任务列表」标题 + 任务数量 Badge。
  - 权重校验提示：`权重: {totalWeight}%`。当 `totalWeight === 100` 时绿色背景 `bg-green-50 text-green-600`；否则黄色背景 `bg-amber-50 text-amber-600`，提示需调整。
- **右侧按钮**：
  1. **添加任务**（主按钮，图标 `Plus`）：打开「添加任务」弹窗。
  2. **克隆/引用**（`variant="outline"`，图标 `Copy`）：打开「克隆/引用任务」弹窗。
  3. **配置任务权重**（`variant="outline"`，图标 `PieChart`）：打开「权重配置」弹窗。

#### 4.3.4 任务列表展示

采用横向表格布局，支持水平滚动（`overflow-x-auto`）：

- **列头行**：8 个配置卡片的标题横向排列，每个标题宽度固定 `w-52 shrink-0`，居中对齐，文字 `text-xs text-gray-500`。
- **任务行**：每个任务占一行，从左到右：
  - **序号区**（`w-8`）：拖拽图标 `GripVertical`（`cursor-grab`）+ 序号数字。
  - **8 个配置卡片**（每个 `w-52 h-36 shrink-0`）：
    - 卡片为 `button` 元素，可点击打开编辑弹窗。
    - 卡片头部：图标（`bg-primary/10 text-primary` 或 `bg-gray-100 text-gray-400`）+ 卡片标题 + 「引用」Badge（仅引用任务显示）。
    - 卡片内容：配置摘要文本（`text-xs line-clamp-5 whitespace-pre-line`），已配置为 `text-gray-600`，未配置为 `text-gray-400`。
    - 引用任务（`isReferenced === true`）的所有卡片（除 weight 外）置灰 `bg-gray-100 opacity-60 cursor-not-allowed`，不可点击。
    - `weight` 卡片为只读展示，不可点击。
  - **删除按钮**（`opacity-0 group-hover:opacity-100`，图标 `Trash2`）。

**8 个配置卡片定义**：

| 卡片类型 | 标题 | 图标 | 配置摘要规则 |
|---------|------|------|-------------|
| info | 配置任务基础信息 | `FileText` | 展示任务名称、编码、类型、难度、预估学时 |
| description | 配置任务说明 | `Book` | 展示描述前50字，未填写展示「未填写」 |
| knowledge | 考查知识点 | `Lightbulb` | 展示已选知识点名称（最多3个），未选展示「未配置」 |
| ability | 考查能力点 | `Award` | 展示已选能力点名称（最多3个），未选展示「未配置」 |
| resources | 配置任务资源 | `Link2` | 展示已选资源名称（最多3个），未选展示「未配置」 |
| evaluation | 配置任务测评形式 | `CheckCircle2` | 展示已选测评形式中文名（顿号分隔），未选展示「未配置」 |
| evaluationRules | 配置任务评价规则 | `Gavel` | 展示各测评形式权重占比，以及「待配置」/「已配置」状态 |
| weight | 配置任务权重 | `Scale` | 只读展示 `{weight}%` |

#### 4.3.5 添加任务弹窗

- 宽度：`sm:max-w-md`。
- 字段：
  - 任务名称（`Input`，必填）。
  - 任务类型（`Select`）：训练任务 / 考核任务。
  - 预估学时（`Input type="number"`），附带说明文字「学生完成任务的预估时长」。
  - 难度（星级选择器，1-5星）。
  - 背景介绍（`Textarea`，3行）。
- 操作：「取消」/「添加」（名称必填）。
- 新增任务后，自动分配默认权重（平均分配）。

#### 4.3.6 克隆/引用任务弹窗

- 宽度：`sm:max-w-4xl`，高度 `max-h-[85vh]`。
- 顶部：模式切换按钮「克隆（可编辑）」/「引用（只读）」。
- 搜索框：按任务名称、编码、关联场景搜索。
- Tab：我的 / 共建 / 公共库。
- 列表：表格展示（任务名称、编码、关联场景、关联岗位），点击行切换复选框选中态。
- 操作：「取消」/「确定 ({selectedCount})」。

#### 4.3.7 权重配置弹窗

- 弹窗内展示所有任务的权重输入框 + 锁定开关（`Switch`）。
- 「均匀分配」按钮：按未锁定任务平均分配权重，确保总和为100%。
- 实时展示总权重，不等于100%时红色提示。

### 4.4 场景编辑器 — 任务配置卡片（8 个步骤）

点击任务行中的配置卡片后，打开「编辑卡片」弹窗（`Dialog`）。弹窗宽度根据内容自适应，通常为 `sm:max-w-4xl` 或 `sm:max-w-[800px]`，高度 `max-h-[90vh]`。

弹窗通用结构：
- **头部**：`DialogTitle` 为卡片标题，`DialogDescription` 为简短说明。
- **主体**：根据卡片类型展示不同表单内容，支持垂直滚动。
- **底部**：「取消」/「保存」。

以下分别描述 8 个卡片的详细交互。

#### 4.4.1 配置任务基础信息（info）

- 任务名称（`Input`）。
- 任务类型（`Select`）：训练 / 考核。
- 预估学时（`Input type="number"`），附带说明文案。
- 难度（星级选择器，1-5星）。
- 背景（`Textarea`，3行）。

#### 4.4.2 配置任务说明（description）

- **Tab 切换**：`富文本编辑` / `上传任务说明书`。
- **富文本编辑**：
  - 工具栏（`bg-gray-50 border-b`）：含标题（H1/H2）、正文、加粗(B)、斜体(I)、下划线(U)、删除线、左/中/右对齐、无序/有序列表、引用、代码、链接、图片、视频、表格、分割线、字体颜色、背景色。
  - 编辑区：`Textarea`，`border-0`，带默认模板文案（任务描述/任务目标/任务结果/测评要求）。
  - 状态栏（`bg-gray-50 border-t`）：左侧「纯文本模式」提示，右侧字符计数。
  - 检测到多媒体内容（`<img` 或 `<video`）时，底部展示蓝色提示条「检测到已插入多媒体内容」。
- **PDF 上传**：
  - 虚线边框上传区，支持点击/拖拽。
  - 已上传后展示 PDF 图标 + 文件名 + 「移除文件」按钮。

#### 4.4.3 考查知识点（knowledge）

采用左右分栏布局（左 3/5，右 2/5）。

- **顶部搜索栏**：搜索知识点名称/描述/编码，右侧「新增知识点」按钮。
- **左侧（知识点库列表）**：
  - 表格表头：知识点名称、编码、描述、操作。
  - 表格行：名称 + 编码 Badge + 描述（`line-clamp-1`）+ 操作按钮组。
  - 操作按钮：
    - 「详情」：打开知识点详情弹窗。
    - 未选中时：「引用」（加入右侧已选）、「克隆」（基于该知识点创建副本并加入已选）。
    - 已选中时：「取消」（从右侧移除）。
  - 搜索无结果时，展示提示「未找到 XXX 相关的知识点」+「新增此知识点」按钮。
- **右侧（已选知识点）**：
  - 标题：已选择知识点 ({count})。
  - 网格卡片（`grid-cols-2 gap-2`）：每张卡片展示名称、描述（`line-clamp-1`）、关联颗粒课 Tag（最多2个，超出显示 `+N`）。
  - 引用知识点：右下角灰色角标「引用」；自定义知识点：无角标，点击打开编辑弹窗。
  - 每张卡片右上角有删除按钮（`X`）。

**知识点详情弹窗**：
- 展示：名称、知识点描述、编码、关联颗粒课列表。
- 引用知识点标注「引用（不可编辑）」Badge；自定义知识点标注「自定义（可编辑）」Badge，并支持「引用颗粒课」「新增颗粒课」按钮。

**新增/克隆/编辑知识点弹窗**：
- 宽度：`sm:max-w-lg`。
- 字段：知识点名称、描述、编码（新增/克隆时自动生成不可改，编辑时可改）、关联颗粒课（Tag 展示 + 「选择颗粒课」按钮 + 「新建颗粒课」外链按钮）。
- 操作：「取消」/「新增并选中」或「克隆并选中」或「保存修改」。

**颗粒课选择弹窗**：
- 宽度：`sm:max-w-[800px]`，高度 `h-[80vh]`。
- 左右分栏：左侧全部颗粒课列表（搜索+复选框），右侧已选列表。
- 操作：「确定」。

#### 4.4.4 考查能力点（ability）

- 若场景未关联岗位，展示警告提示「请先关联岗位后，再选择考察能力点」。
- **顶部搜索栏**：搜索能力点名称/编码/描述，右侧展示统计「共 N 个关联能力点，已选 M 个」。
- **能力点分组展示**：按「能力领域（domain）」分组卡片。
  - 每个分组为可折叠卡片，头部：领域图标 + 领域名 + 能力点数量 Badge。
  - 展开后：能力点列表，每行含复选框、名称、编码、掌握程度 Badge（了解/理解/掌握/熟练/精通，各有颜色）、描述（`line-clamp-1`）、关联岗位 Tag。
  - 点击整行切换选中态。已选中行左侧带 `border-l-2 border-l-primary` 高亮。
- 搜索无结果时展示空态图标+提示。
- **能力点详情弹窗**：展示名称、描述、所属领域、关联岗位、掌握程度要求、熟练程度描述。

> 能力点数据来源于系统功能能力库，不从本模块维护。

#### 4.4.5 配置任务资源（resources）

采用左右分栏布局（左自适应，右 `w-72`）。

- **顶部工具栏**：
  - 资源类型筛选标签（横向排列按钮）：全部、文档、表格、图片、链接、音频、视频、压缩包、场地、设施设备、软件、其他。选中态高亮。
  - 搜索框：资源名称、资源提供者，中间夹「重置」按钮。
  - 「上传资源」按钮。
- **左侧（资源库网格）**：
  - 网格卡片（`grid-cols-2 md:grid-cols-3 lg:grid-cols-4`）。
  - 每张卡片：顶部缩略图区（文件类型图标+类型 Badge）+ 选中角标（`CheckCircle2`）+ 信息区（名称、提供者、日期）+ 底部操作区（「预览」外链按钮、「选择/取消」按钮）。
  - 点击卡片非按钮区域也切换选中态。
- **右侧（已选资源列表）**：
  - 标题 + 数量 Badge。
  - 列表项：类型图标 + 名称 + 提供者·日期 + 删除按钮。
  - 空态：提示「请从左侧选择资源」。

**上传资源类型选择弹窗**：
- 网格展示 11 种资源类型图标+名称，点击后进入上传弹窗并预填类型。

**上传资源弹窗**：
- 宽度：`sm:max-w-md`。
- 通用字段：资源名称（必填）、资源描述（`Textarea`）。
- 根据资源类型展示特定字段：
  - **链接**：URL 地址（必填）。
  - **场地**：场地地址、开放时间、容纳人数、联系人/电话。
  - **设施设备**：所在位置、数量。
  - **软件**：版本号、下载链接、授权信息。
  - **文件类**（文档/表格/图片/音频/视频/压缩包/其他）：文件上传拖拽区（提示「支持多种格式，最大 100MB」）。
- 操作：「取消」/「上传并选中」（名称必填，链接类型时 URL 必填）。

#### 4.4.6 配置任务测评形式（evaluation）

- 按分类分组展示（基础考核、综合评估、互动评价、智能评测），每组一个浅色背景卡片区。
- 每个测评形式以卡片按钮展示：
  - 左侧：图标 + 名称（`font-semibold`）+ 描述（`text-[11px] text-gray-400`）。
  - 右侧：
    - 「查看介绍」按钮（书本图标，点击新开标签页跳转 `/wiki/eval-method?key={methodKey}`，仅对已开通测评形式显示）。
    - 已开通时：绿色「已开通」Badge。
    - 未开通时：灰色「未开通」Badge，卡片整体 `opacity-50 cursor-not-allowed`，中央盖印「未开通」水印。
  - 点击卡片切换选中/取消选中（仅对已开通有效）。选中态：`border-primary ring-1 ring-primary/20 shadow-sm`。
- 本期仅 4 种测评形式为 `available=true`：**现场问答**、**现场评审**、**试卷**、**题库**。其余均显示未开通状态。
- 底部空态：未选任何测评形式时展示提示「请选择至少一种评价方式」。

#### 4.4.7 配置任务评价规则（evaluationRules）

> 本节为最复杂配置模块，包含评价点管理、评分主体配置、各测评形式的细化规则。

**页面结构**：纵向堆叠多个配置区块。

**A. 测评形式配置概览**

- 对当前任务已选的每种测评形式，展示一个概览卡片：
  - 左侧：测评形式名称 + 配置状态（已配置/待配置，带颜色区分）。
  - 右侧：配置摘要（如「5 题 / 3 个评价点」、「已选择 XX 试卷」）。
  - 点击卡片展开该测评形式的详细配置面板。

**B. 评分主体配置（全局默认）**

- 6 种评分主体，每个主体一行：
  1. **教师**：默认启用。参数：教师专业背景、评分人数、权重占比、评分维度、最低教龄、分数聚合规则（平均/中位数/最高/最低）。
  2. **企业导师**：默认启用。参数：专业领域、最低工作年限、企业类型、岗位经验、评分人数、权重占比。
  3. **自评**：默认启用。参数：是否需要反思、反思最低字数、权重占比。
  4. **互评**：默认禁用。参数：互评人数、互评规则、是否匿名、权重占比。
  5. **AI 评价**：默认禁用。参数：AI 模型、置信度阈值、自动复核、权重占比。
  6. **服务对象**：默认禁用。参数：服务方式、样本量、权重占比。
- 每行使用 `Switch` 控制启用/禁用，启用后展示参数配置区。

**C. 评价点管理**

- 每个测评形式独立维护评价点列表。
- 评价点卡片（`EvalPointCard`）：
  - 名称输入框（可编辑）。
  - 子类型 Badge（知识掌握/操作规范/任务完成度/成果质量/沟通表达/协作能力/职业素养/创新能力/应变能力，各有颜色）。
  - 评分方式选择（`Select`）：本期仅「等级制」可用（分值制和 rubric 量表 disabled）。
  - 关联能力点：展示已关联能力点 Tag（可删除）+「+ 添加能力点」按钮（打开能力点搜索弹窗）。
  - 关联知识点：展示已关联知识点 Tag（可删除）+「+ 添加知识点」按钮（打开知识点搜索弹窗）。
  - 等级转换规则编辑器（`LevelRuleEditor`）：
    - 顶部可视化条带：按分数区间展示 A/B/C/D 色块。
    - 每个等级卡片：等级名称输入（如 A/B/C/D）、颜色圆点、分数区间（min-max 数字输入）、等级备注（一句话辅助教师参考）。
  - 删除按钮（`Trash2`）。
- 新增评价点：底部输入框输入名称后按回车或点击添加。支持从能力点/知识点快速生成评价点。

**D. 各测评形式细化配置**

- **现场问答（random_draw）**：
  - 抽题规则：题目数量、难度分布（混合/简单/中等/困难）、题型范围（单选/多选/判断，复选框）、是否自动抽题。
  - 选题弹窗：题库列表表格，支持搜索，勾选题目加入已选。

- **现场评审（review）**：
  - 评审材料要求：材料类型（项目报告/设计文档/代码/视频等）、提交格式说明、截止天数、是否允许重提交。
  - 评审步骤配置：默认「初评」，可启用「复评」「终评」。每步可配置：步骤名称、描述、评审主体类型、权重。支持新增/编辑/删除步骤。
  - 场地资源说明：文本描述。

- **试卷（paper）**：
  - 试卷选择：从试卷库选择（展示试卷名称、题数、总分）。
  - 新建试卷弹窗：试卷名称、题目数量、总分。
  - 试卷详情查看弹窗。
  - 参数配置：考试时长、及格分数、是否允许重考、重考次数、是否乱序、是否展示成绩、激活方式（手动/定时/永久开放）、定时时间。

- **题库（question_bank）**：
  - 抽题规则：题目数量、难度分布、总分、是否自动生成、时间限制、题型权重、是否允许重考、乱序、展示成绩。
  - 选题弹窗：题库列表，支持搜索，勾选题目。

**E. 评价规则保存逻辑**

点击弹窗「保存」时，将评价点、评审步骤等数据同步回任务对象。评价点需转换为 `TaskEvalPoint` 结构存储。

#### 4.4.8 配置任务权重（weight）

- 在任务列表的 weight 卡片中只读展示百分比。
- 点击「配置任务权重」按钮打开弹窗：
  - 每个任务一行：任务名称 + 权重数字输入框 + 锁定开关（`Lock`/`Unlock` 图标）。
  - 锁定任务在「均匀分配」时权重保持不变。
  - 底部总权重实时计算，不等于 100% 时红色高亮提示。
