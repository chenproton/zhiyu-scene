## 一、测试用例说明

### 1.1 优先级定义

| 优先级 | 定义 | 策略 |
|--------|------|------|
| **P0** | 核心业务流程，阻塞性缺陷 | 必须 100% 通过，回归必测 |
| **P1** | 重要功能，影响用户体验 | 必须 100% 通过 |
| **P2** | 一般功能，异常/边界场景 | 主要路径通过 |
| **P3** | 边缘场景、低频率操作 | 抽样验证 |

### 1.2 用例编号规则

| 前缀 | 含义 |
|------|------|
| `TC-SCENE` | 场景学习页（学生端） |
| `TC-BUILD` | 场景建设（管理端） |
| `TC-APPR` | 审批中心（教师端） |
| `TC-GRADE` | 评分/批阅（教师端） |
| `TC-WF` | 工作流与批次 |
| `TC-OTHER` | 其他模块 |
| `TC-PERF` | 性能测试 |

### 1.3 角色说明

| 角色 | 功能范围 |
|------|----------|
| Student | 场景学习、任务测评、知识图谱浏览 |
| Builder | 场景建设、岗位管理 |
| Reviewer | 审批、评分 |
| Admin | 全权限（本轮回归测试中视同 Builder + Reviewer） |

| `GET /api/v1/grading/submissions` | GET | 查询学生提交列表（教师评分） |

### 5.3 用例执行建议

1. **冒烟测试**：每次发版前执行全部 P0 用例（TC-SCENE-001 ~ TC-SCENE-020，TC-BUILD-001 ~ TC-BUILD-045 中的全部 P0 项，TC-APPR-001 ~ TC-APPR-003，TC-GRADE-001 ~ TC-GRADE-005，TC-PERF-001 ~ TC-PERF-002）
2. **回归测试**：功能迭代后执行对应模块 P0 + P1 用例
3. **性能基线**：每月执行一次完整性能测试套件，与历史数据对比
