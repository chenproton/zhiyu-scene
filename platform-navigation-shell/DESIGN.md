# Platform Navigation System - 设计与视觉基线

这份文档用于给集成了 `platform-navigation-shell` 的系统提供一套统一的中层设计基线。

## 1. 技术上下文

推荐以下组合保持一致：

- React / Next.js App Router
- Tailwind CSS
- shadcn/ui 风格组件
- lucide-react 图标

如果宿主系统已有自己的业务组件库，也建议至少在按钮、输入框、弹窗、下拉菜单这类基础交互上保持接近 shadcn/ui 的气质。

## 2. 系统骨架

所有接入系统建议遵守同一套 B 端三层结构。

### 第一层：全局顶导

- 固定高度：`h-14`
- 位置：固定在页面顶部
- 背景：纯白
- 分层方式：`border-b border-gray-100`

建议承载：

- 系统 Logo / 品牌名
- 全局时间
- 用户信息
- 顶部三个全局入口

顶部入口建议保持统一文案风格：

- `门户首页`
- `我的服务台`
- `应用服务中心`

说明：

- 是否可点击可以按宿主系统实际情况决定
- 但顶导结构、节奏和视觉表达建议尽量统一

### 第二层：模块导航

- 形式：左侧边栏
- 作用：承载子模块、子页面、分组入口
- 激活态：使用主色调和指示条强调

建议保留的体验特征：

- 分组清晰
- 当前模块位置明确
- Hover 和 Active 都要有明显反馈

### 第三层：主工作区

- 页面背景：`bg-[#f5f7fa]`
- 内容区避开固定顶导：如 `pt-14`
- 主体内容常用 `max-w-6xl mx-auto`

可根据业务场景微调为：

- `max-w-4xl`
- `max-w-5xl`
- `max-w-6xl`

重点不是所有页面宽度完全一致，而是整体内容区不要失控铺满。

## 3. 视觉底座

### 主色

统一使用：

- `bg-primary`
- `text-primary`

主色主要用于：

- 当前导航激活
- 主按钮
- 关键图标容器
- 强调型数字或链接

### 容器层级

主要内容容器建议使用：

- `bg-white`
- `border border-gray-100`

整体更依赖细边框分层，而不是厚重阴影。

可以有轻微阴影，但不建议让阴影成为主要层级手段。

### 圆角

建议基线：

- 大卡片、主要面板、图标容器：`rounded-xl`
- 常规按钮、输入框、小型控件：`rounded-md` 或 `rounded-lg`
- 头像、状态点：`rounded-full`

这不是强制尺寸表，而是建议保持同一类组件的圆角气质一致。

## 4. 文字层级

建议统一文字语义：

- 页面级大标题：`text-3xl font-bold`
- 卡片或区块标题：`text-sm font-medium leading-tight`
- 正文说明：`text-sm text-gray-500`
- 更轻的辅助信息：`text-xs` 或 `text-sm text-gray-400`

推荐颜色分工：

- 主标题 / 强调文字：`text-gray-800`
- 正文 / 常规描述：`text-gray-500`
- 辅助 / 次要信息：`text-gray-400`

说明：

- 各系统可以根据密度调整字号
- 但标题、正文、辅助信息之间的层级关系应保持一致

## 5. 语义色使用方式

当需要多色区分时，推荐使用“浅底 + 同色文字 + 超细边框”的方式，而不是高饱和纯色块。

例如：

- `bg-blue-50 text-blue-500 border border-blue-100`
- `bg-rose-50 text-rose-500 border border-rose-100`
- `bg-amber-50 text-amber-600 border border-amber-100`
- `bg-green-50 text-green-600 border border-green-100`

适合用在：

- Badge
- 小图标底座
- 分类标签
- 状态提示

这样可以让页面有足够的信息区分度，同时不会破坏整体后台气质。

## 6. 交互态基线

### 导航 Hover

建议使用：

- `hover:text-gray-800`
- `hover:bg-gray-50`

### 幽灵按钮 / 透明按钮 Hover

建议使用：

- `hover:text-primary`
- `hover:bg-primary/5`

### 当前激活导航

建议使用主色指示条，例如：

- 底部或侧边添加 `h-0.5 bg-primary rounded-full`

### 过渡

建议保留轻微过渡，让页面显得更精致：

- `transition-all`
- `transition-colors`

不建议做过重、过长、过花的动效。

## 7. 自由发挥的部分

在遵守上面这些基线后，下面的内容可以根据业务自由设计：

- 卡片具体长什么样
- 首页模块怎么排
- Grid 还是 Flex
- 表格还是卡片列表
- 图表长什么样
- 双栏还是单栏
- 筛选器放顶部、左侧还是卡片内

也就是说：

- 骨架和气质统一
- 业务内容自由组织

## 8. 接入时的使用建议

如果你要把这套壳子迁移到另一个现有系统，建议按这个顺序统一：

1. 先接入导航壳
2. 再统一工作区背景、标题层级、主色使用方式
3. 再统一状态色和基础 Hover 反馈
4. 最后根据业务特性保留页面自由度

## 9. 参考理解方式

请把这份文档理解为：

- 不是页面模板
- 不是组件尺寸手册
- 不是逐像素规范

它更像是一份“统一家族风格的设计提示词”。

后续集成新系统时，判断标准很简单：

- 用户切换到另一个系统时，是否还能感受到它们属于同一个平台体系
- 即使模块布局不同，是否仍然拥有一致的品牌感、阅读感和后台感
