# 测评方式功能变更文档

**对比版本：** `d87dca7` → 当前工作区（Working Tree）  
**主要影响文件：**
- `app/scenarios/[id]/edit/tasks/page.tsx`
- `public/student_teacher.html`

---

## 一、配置任务评价规则弹窗（evaluationRules）

### 1. 测评方式支持复制（核心新增）

**变更前：**
- 每种测评方式仅展示一行（如：题库、试卷、随堂测等）。

**变更后：**
- 每种测评方式卡片右上角新增「复制测评方式」按钮（图标 + 文案）。
- 点击后弹出浏览器 `confirm` 二次确认：`确定要复制「XXX」测评方式吗？`
- 确认后在同种测评方式下新增一行，名称自动加序号，例如：
  - 第一次复制：「题库」→「题库 2」
  - 第二次复制：「题库 2」→「题库 3」
- 复制的行包含与原行相同的四列配置入口：测评对象、评价主体、测评资源、评价标准配置。

**实现方式：**
- 新增 `methodInstanceCounts` 状态（纯 UI 模拟，不持久化到 TaskState）。
- 新增 `getMethodInstances()` 生成带 `instanceIndex` 的实例列表。
- 新增 `duplicateMethod(methodKey)` 递增计数。
- 原 `state.evaluationMethods.map(...)` 渲染改为 `getMethodInstances().map(...)`。

### 2. 评价顺序/权重弹窗同步展示复制实例

**变更前：**
- 「配置评价顺序」和「配置评价权重」弹窗中仅展示原始测评方式列表。

**变更后：**
- 两个弹窗均基于 `getMethodInstances()` 渲染。
- 复制的实例也会出现在顺序和权重配置中，显示为「题库 2」「随堂测 2」等。

## 二、QuestionSelectorPanel 组件增强

### 1. 新增 `showAutoSelect` 属性

**用途：** 控制是否在「已选择题目」右上角展示「自动选择」按钮。

**行为：**
- 点击后 `alert("参考测评认证中心-试卷管理-自动抽题功能即可")`。
- 题库（`question_bank`）和随堂测（`quiz`）已开启该属性。

### 2. 新增 `maxCount` 属性

**用途：** 在「已选择题目」标题后显示数量上限，例如 `已选择题目 (3/30)`。

**应用：**
- 随堂测设置了 `maxCount={30}`，显示为 `已选择题目 (x/30)`。
- 题库未设置，显示为 `已选择题目 (x)`。

## 三、各测评方式的详细变更

### 3.1 现场问答（random_draw）

#### 3.1.1 适用专业筛选 Tab

**变更前：**
- 左侧题目列表上方仅有搜索框，无专业分类。
- 新增/编辑题目时的专业下拉选项为技术类：前端开发、后端开发、运维部署、通用。

**变更后：**
- 左侧列表上方新增适用专业 Tab 切换条：
  **全部、经济学、物流管理、机械工程、计算机科学、电子信息、工商管理、会计学、市场营销、土木工程、英语、法学**
- 默认选中「全部」。
- 题目列表按所选专业实时过滤。
- 新增/编辑弹窗中的专业下拉同步为上述学校专业名称。

#### 3.1.2 新增「抽题规则」模块

在弹窗底部新增「抽题规则」卡片，包含两个字段：

| 字段 | 类型 | 选项/说明 |
|------|------|----------|
| 抽题方式 | 下拉选择 | 系统随机分配 / 老师手动选择 |
| 抽题数量 | 数字输入框 | 最小值为 1 |

---

### 3.2 题库（question_bank）

#### 3.2.1 自动选择按钮

- 「已选择题目」右上角新增「自动选择」按钮，点击提示参考测评认证中心自动抽题功能。

#### 3.2.2 抽题规则 → 答题规则

**文案变更：**
- 模块标题：「抽题规则」→「答题规则」
- 字段标题：「抽题方式」→「答题方式」

#### 3.2.3 答题方式条件展示

**全部作答（默认）：**
- 时间限制（分钟）
- 允许重复测评
- 题目乱序
- 提交后展示成绩
- 若允许重复测评，展示「最多重考次数」

**自由刷题：**
- 在全部作答字段基础上新增：
  - **正确率（%）**：输入框，范围 0-100
  - 说明文案：「超过正确率则得分 100，低于正确率得分 0」

**移除字段：**
- 随机抽题数量
- 难度分布

---

### 3.3 随堂测（quiz）

#### 3.3.1 自动选择按钮

- 与题库一致，「已选择题目」右上角新增「自动选择」按钮。

#### 3.3.2 题目数量上限

- 已选题目显示 `已选择题目 (x/30)`，提示最多选择 30 道题。

#### 3.3.3 抽题规则 → 答题规则

- 模块标题由「抽题规则」改为「答题规则」。

#### 3.3.4 时间限制改为单选块 + 自定义

**变更前：**
- 时间限制为一个数字输入框。

**变更后：**
- 时间限制改为按钮单选：5 分钟、10 分钟、15 分钟、20 分钟、30 分钟。
- 新增「自定义」按钮，点击后下方显示数字输入框，可自由输入分钟数。
- 选中自定义时，输入框值为当前 timeLimit；若 timeLimit 不在预设列表中，自动进入自定义模式。

#### 3.3.5 移除字段

- 随机抽题数量
- 难度分布

---

### 3.4 试卷（paper）

#### 3.4.1 试卷选择改为单选

**变更前：**
- 「选择已有试卷（可多选）」，复选框多选。
- 多选后下方出现「试卷权重分配」模块，需手动分配各试卷权重百分比。

**变更后：**
- 文案改为「选择已有试卷」。
- 选择控件由复选框改为圆形单选按钮。
- 选择某试卷后自动设为 100% 权重，隐藏原「试卷权重分配」模块。

#### 3.4.2 试卷启用条件文案优化

三种启用方式的描述文案补充为完整业务说明：

| 启用方式 | 描述文案 |
|---------|---------|
| 后台手动启用 | 老师手动开启试卷后，学生才能进入作答。每位学生从点击「开始答题」时起，计时 {duration} 分钟，时间结束系统将自动提交试卷。 |
| 定时启用 | 提前预设考试的开始、结束时间。到开始时间后，学生方可进入作答；从开始时间起按考试时长计时，到预设结束时间自动关闭并提交试卷。 |
| 随时作答 | 试卷创建完成后立即开放，学生可随时进入作答。学生从点击「开始答题」时起按试卷设定时长计时，时间结束系统将自动提交试卷。 |

#### 3.4.3 定时启用增加停用时间

**变更前：**
- 仅有一个「启用时间」字段。

**变更后：**
- 选择「定时启用」后，展示两个字段：
  - **启用时间**
  - **停用时间**

#### 3.4.4 启用条件样式调整

- 由横向按钮排列改为纵向卡片列表。
- 每个选项左侧增加圆形单选图标，下方展示描述文案。

## 四、其他说明

### 数据持久化

本轮所有新增功能均为 **UI 模拟效果**，刷新页面后：
- 测评方式复制数量会重置为 1。
- 现场问答的抽题规则会重置为默认值（系统随机分配，数量 5）。
- 题库的答题方式会重置为「全部作答」。
- 随堂测的时间限制会重置为默认值。
- 试卷的停用时间会清空。

### 未变更项

以下测评方式未做调整，保持 `d87dca7` 版本逻辑：
- 现场评审（review）
- 成果评价（outcome）
- 作业（homework）

### 静态 HTML 版本（public/student_teacher.html）

该文件在 `d87dca7` 之后也有修改，主要用于独立的静态演示页面。变更内容包括：
- 测评资源配置弹窗中每种测评方式支持多份配置（`ecRandomDrawItems` 等数组）。
- 配置列表 UI、复制按钮、自动序号命名等逻辑。

> 注：本次用户主要验证和使用的是 Next.js 应用页面（`/scenarios/[id]/edit/tasks`），`public/student_teacher.html` 为早期独立演示文件。
